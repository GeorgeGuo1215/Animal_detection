## `/v1/chat/completions` 接口文档

### 基本信息

| 项目 | 值 |
|------|-----|
| 路径 | `POST /v1/chat/completions` |
| 协议 | OpenAI 兼容格式 |
| 鉴权 | `Authorization: Bearer <api-key>` |
| Content-Type | `application/json` |

### 请求体

```json
{
  "model": "agent-plan-solve",
  "messages": [
    {"role": "system", "content": "可选的系统提示词"},
    {"role": "user", "content": "用户问题"}
  ],
  "stream": true,
  "temperature": 0.7,
  "max_tokens": 1500,
  "tools": [{"function": {"name": "rag.search"}}]
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `model` | string | 否 | `agent-plan-solve` | 模型/Agent 模式（见下方） |
| `messages` | array | **是** | - | OpenAI 格式消息数组 |
| `stream` | bool | 否 | `false` | 是否流式返回（SSE） |
| `temperature` | float | 否 | `0.2` | 生成温度 |
| `max_tokens` | int | 否 | `768` | 最大生成 token 数 |
| `tools` | array | 否 | 全部可用工具 | 限制 Agent 可调用的工具列表 |
| `debug_timing` | bool | 否 | `true` | 是否返回各步骤耗时 |

### model 可选值

| model | 行为 |
|-------|------|
| `agent-plan-solve` | 先让 LLM 规划步骤（JSON），再依次执行工具，最后综合生成回答。适合单轮复杂问题 |
| `agent-multi-turn` | 多轮迭代决策（最多 5 轮），每轮 LLM 自主决定调用工具或生成最终回答。适合需要多步推理的场景 |

### tools 格式

不传 `tools` 时，默认启用所有已注册工具。若需限制，传入工具名列表：

```json
"tools": [
  {"function": {"name": "rag.search"}},
  {"function": {"name": "mcp.web_search.web_search"}},
  {"function": {"name": "mcp.nutritional_planner.calculate_meal_plan"}}
]
```

可用工具名：`rag.search`、`mcp.web_search.web_search`、`mcp.web_search.ingredient_check`、`mcp.nutritional_planner.calculate_meal_plan`、`mcp.nutritional_planner.generate_exercise_plan`

### 非流式响应（`stream: false`）

```json
{
  "id": "chatcmpl-abc123...",
  "object": "chat.completion",
  "created": 1700000000,
  "model": "agent-plan-solve",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "根据兽医知识库的信息，您的狗食欲不振可能有以下原因..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0},
  "plan": [
    {"type": "tool", "tool_name": "rag.search", "arguments": {"query": "..."}},
    {"type": "final"}
  ],
  "tool_results": [
    {"step": 0, "tool_name": "rag.search", "arguments": {...}, "result": {...}}
  ],
  "timing": [
    {"step": "plan_llm", "ms": 1200},
    {"step": "solve_all", "ms": 3500},
    {"step": "total", "ms": 4700}
  ]
}
```

### 流式响应（`stream: true`）

返回 SSE（`text/event-stream`），每条消息格式为 `data: {JSON}\n\n`，最后以 `data: [DONE]\n\n` 结束。

每个 chunk 结构：

```json
{
  "id": "chatcmpl-abc123...",
  "object": "chat.completion.chunk",
  "created": 1700000000,
  "model": "agent-plan-solve",
  "choices": [{"index": 0, "delta": {"content": "文本片段"}, "finish_reason": null}],
  "agent_status": "streaming",
  "agent_detail": null
}
```

`agent_status` 状态流转：

| 状态 | 含义 |
|------|------|
| `planning` | 正在规划步骤（仅 plan-and-solve） |
| `plan_complete` | 计划制定完成 |
| `thinking` | 正在思考/决策（仅 multi-turn） |
| `tool_calling` | 正在调用工具 |
| `tool_complete` | 工具调用完成 |
| `user_action_needed` | 需要用户提供额外信息 |
| `generating` | 开始生成最终回答 |
| `streaming` | 正在流式输出回答文本 |
| `decided_final` | 决定生成最终回答（仅 multi-turn） |
| `timing_summary` | 耗时统计（debug_timing=true） |

### curl 示例

**非流式：**

```bash
curl -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-pethealthai-default-key-2026" \
  -d '{
    "model": "agent-plan-solve",
    "messages": [{"role": "user", "content": "我的狗最近食欲不振，该怎么办？"}],
    "stream": false
  }'
```

**流式：**

```bash
curl -N -X POST http://localhost:8000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-pethealthai-default-key-2026" \
  -d '{
    "model": "agent-multi-turn",
    "messages": [{"role": "user", "content": "10kg成年犬今天该吃多少克狗粮？"}],
    "stream": true,
    "temperature": 0.7,
    "max_tokens": 1500
  }'
```

### 错误响应

```json
{
  "error": {
    "message": "Missing API key: set OPENAI_API_KEY (or DEEPSEEK_API_KEY).",
    "type": "server_error",
    "code": "agent_error"
  }
}
```

HTTP 状态码：`401`（鉴权失败）、`429`（限流）、`500`（Agent 内部错误）
